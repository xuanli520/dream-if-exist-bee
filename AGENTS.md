# AGENTS.md

## 项目类型

- 这是原生微信小程序项目，不是 `uni-app`、`Taro`、`Vue` 或 `React` 项目。
- 应用入口是 `app.js`，页面注册和全局组件声明在 `app.json`。
- 远程接口通过 `apifm-wxapi` 调用，基础接入配置在 `config.js`。

## 工作流程

- 严格按 `Explore -> Plan -> Implement` 执行。
- 先读当前仓库代码和配置，再动手修改。
- 只做用户明确要求的改动，保持变更聚焦。
- 所有文件按 UTF-8 读取和写入。

## 目录约束

- `pages/`：页面目录，通常一组页面由 `.js`、`.json`、`.wxml`、`.wxss` 组成。
- `components/`：复用组件。
- `utils/`：认证、支付、图片、工具类、运行时兼容逻辑。
- `i18n/`：多语言资源和底部菜单文案。
- `images/`：静态资源。
- `miniprogram_npm/`：已打包依赖，除非用户明确要求，不要直接修改。

## 修改规则

- 新增、删除、移动页面时，必须同步更新 `app.json`。
- 修改底部菜单相关内容时，同时检查 `app.json` 与 `i18n/index.js`。
- 修改页面文案时，同时检查 `i18n/zh_CN.js` 和 `i18n/en.js`。
- 涉及支付、登录、会员卡、门店、配送、地图、客服时，先核对 `config.js` 和 `app.js` 中的远程配置读取逻辑。
- 现有接口调用风格以 `WXAPI` 为准，除非用户明确要求，不要自行重构整套调用方式。
- 不要把 `project.private.config.json` 当作团队公共配置文件。
- 不要沿用上游模板的宣传文案或外部教程描述，文档和实现都以当前仓库实际代码为准。

## 运行认知

- 这个仓库依赖微信运行环境和远程后台，不是纯本地离线项目。
- `config.js` 中的 `subDomain`、`merchantId` 会直接影响接口可用性。
- `app.js` 启动时会批量拉取远程参数，很多页面行为依赖这些缓存值。
- 扫码点餐、自提、外卖、提货点、企业微信客服等能力都和后台配置有关。

## 输出要求

- 与用户交流使用中文。
- 默认不要添加注释、文档、测试或额外功能，除非用户明确要求。
- 不要做无关重构，不要扩大修改范围。
